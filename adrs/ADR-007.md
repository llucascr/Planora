# ADR-007: Geração de Backlog via Chamada Síncrona à API Python

## Status

Substituído pelo [ADR-008](ADR-008.md)

## Contexto

Quando implementamos a integração com o serviço Python de IA, a abordagem mais direta era fazer uma chamada HTTP síncrona ao endpoint /api/v1/generate-backlog e aguardar o retorno com as issues geradas. Essa foi a primeira implementação, realizada no início da integração com a IA (ApiPythonClient e ApiPythonService iniciais).

## Decisão

O ApiPythonClient declarava um único método generateBacklog() que fazia um POST e retornava AcceptedResponse diretamente. O controller chamava apiPythonService.generateBacklog(description) e retornava o resultado ao frontend na mesma requisição HTTP.

## Consequências

### Positivas

- Implementação simples e direta, sem necessidade de polling ou estado persistido.
- Fácil de testar localmente com chamadas únicas.

### Negativas / Trade-offs

- O modelo Llama 3.1 8B leva de 30 a 90 segundos para gerar o backlog, estourando o timeout HTTP.
- O frontend ficava bloqueado aguardando a resposta, sem feedback de progresso para o usuário.
- Qualquer falha de rede no meio da geração perderia todo o resultado sem possibilidade de recuperação.
