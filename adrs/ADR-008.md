# ADR-008: Geração de Backlog via Sistema de Jobs com Callback Assíncrono

## Status

Aceito (substitui [ADR-007](ADR-007.md))

## Contexto

A abordagem síncrona do ADR-007 falhou em ambiente real porque o modelo Llama 3.1 8B leva mais tempo do que o timeout HTTP padrão (de 30 a 90 segundos por geração). Precisávamos de uma forma de iniciar a geração, liberar a conexão HTTP imediatamente, e entregar o resultado quando ficasse pronto, sem perder o estado da requisição.

## Decisão

Ao receber a requisição do frontend (POST /v1/ia), o backend:

1. Persiste um registro Job no PostgreSQL com os parâmetros da requisição (boardId, columnId, userId, repository, description).
2. Chama de forma síncrona o endpoint POST /api/v1/generate-backlog do serviço Python, passando apenas o jobId e a descrição. Se a conexão falhar, retenta até 5 vezes com intervalo de 3 segundos.
3. Retorna 200 OK ao frontend imediatamente após a confirmação de recebimento pelo Python.

O serviço Python processa o backlog de forma independente (via Ollama + Llama 3.1 8B), valida o JSON gerado e, ao terminar, chama de volta o endpoint POST /v1/ia/callback do backend com as issues geradas. O backend salva as issues diretamente no board Kanban correspondente usando o jobId para recuperar os parâmetros originais.

## Consequências

### Positivas

- O frontend recebe resposta imediata e continua responsivo enquanto a IA processa o backlog.
- As issues são salvas diretamente no board, eliminando perda por falha de rede durante a transmissão.
- O retry automático (até 5 tentativas) absorve alucinações de formato do modelo sem intervenção do usuário.
- O serviço Python pode ser reiniciado ou escalado de forma independente do backend.

### Negativas / Trade-offs

- Adiciona uma dependência externa que pode falhar de forma independente do backend.
- O deploy requer orquestração dos dois serviços juntos, o que é feito com Docker Compose.
- Se o callback falhar (falha de rede, bug no serviço Python), as issues geradas não chegam ao board e o erro é silencioso para o usuário.
