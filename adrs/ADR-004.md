# ADR-004: Cliente GitHub como Interface Declarativa com Spring HTTP Exchange

## Status

Aceito

## Contexto

Precisávamos chamar vários endpoints da API do GitHub. Escrever chamadas manuais com RestTemplate ou WebClient para cada endpoint geraria muito código repetitivo e difícil de manter.

## Decisão

Definir o GithubClient como uma interface Java anotada com @GetExchange, @PostExchange e similares. O Spring 6 gera a implementação automaticamente em tempo de execução via proxy. A configuração da URL base e dos timeouts fica centralizada em GithubClientConfig.

## Consequências

### Positivas

- Código limpo e declarativo: cada método mapeia diretamente para um endpoint da API.
- Fácil de substituir por um mock nos testes unitários.
- Configuração centralizada facilita mudanças na autenticação ou na URL base.

### Negativas / Trade-offs

- A interface acabou ficando grande demais, com dez métodos de seis domínios diferentes, o que viola o Princípio da Segregação de Interfaces (ISP). A refatoração planejada prevê dividir GithubClient em interfaces menores por domínio: GithubIssueClient, GithubRepositoryClient e GithubWebhookClient.
