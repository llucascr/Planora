# ADR-001: Uso de PostgreSQL como banco de dados relacional

## Status

Aceito

## Contexto

Precisávamos de um banco que suportasse bem os relacionamentos entre as entidades do sistema, como usuários, quadros Kanban, colunas, membros e issues. A equipe já tinha experiência com bancos relacionais e o Spring Data JPA funciona muito bem com PostgreSQL.

## Decisão

Usar PostgreSQL como único banco de dados do sistema, acessado via Spring Data JPA com Hibernate. As entidades são mapeadas com anotações JPA padrão.

## Consequências

### Positivas

- Garante consistência transacional com suporte a ACID.
- Integração nativa e simples com Spring Boot.
- Suportado pelas principais plataformas de deploy na nuvem.

### Negativas / Trade-offs

- Mudanças no schema exigem migrações cuidadosas.
- Escalar horizontalmente requer estratégias adicionais como réplicas de leitura.
