# ADR-006: Identificação de Repositório no Formato owner/repository

## Status

Aceito (substitui formato anterior que aceitava apenas o nome do repositório)

## Contexto

No início, o sistema aceitava apenas o nome do repositório ao criar um KanbanBoard. Isso funcionava para repositórios pessoais do usuário, mas impedia a criação de boards vinculados a repositórios de organizações ou de outros usuários dos quais o membro faz parte. Percebemos essa limitação quando integrantes tentaram usar repositórios de organizações da PUC durante o desenvolvimento.

## Decisão

O campo de repositório passou a aceitar o formato owner/repository, onde owner é o nome do usuário ou organização dona do repositório. O KanbanBoardService faz o parse da string separando pelo caractere "/" e usa os dois valores separados para todas as chamadas à API do GitHub. O formato é validado na criação do board, lançando IllegalArgumentException se o formato estiver incorreto.

## Consequências

### Positivas

- Suporta repositórios de organizações e de outros usuários, não apenas repositórios pessoais.
- O par owner/repository é o identificador canônico da API do GitHub, alinhando o modelo de dados com a API.
- A validação explícita do formato evita erros silenciosos nas chamadas à API do GitHub.

### Negativas / Trade-offs

- O campo precisa ser preenchido no formato correto, o que exige orientação ao usuário no frontend.
- O owner precisa ser recuperado e armazenado separadamente de outras informações do repositório.
